# Used: 2014-11-08 - TRAFFIC ANALYSIS EXERCISE: QUESTIONS INVOLVING EXPLOIT KIT (EK) ACTIVITY

## LEVEL 1 QUESTIONS:

### 1. What is the IP address of the workstation (the VM) that gets infected?\
Answer: `172.16.165.133`
Steps:
- Went to `Statistics > Protocol Hierarchy`
- Saw Data was *6.6%* of packets
- Applied as filter
- Found the string `"This program cannot be run in DOS mode"`
- Copied the destination IP

### 2. Search the nbns traffic in the pcap to find the host name of the infected VM.\
Answer: `WIN-C2KE6N4W3N1`
Steps:
- Exported the specified target packets `ip.addr == 172.16.165.133`
- Used the `nbns` filter to display the packets and got the name

### 3. What is the MAC address of the infected VM?\
Answer: `00:0c:29:77:ac:27`
Steps:
- Copied the MAC from the Packet detail tab (`Ethernet II`)

### 4. What is the IP address and domain name that delivered the malware?
Answer: `seedradrivergy.co.vu` (`178.63.209.91`)
Steps:
- Search `ip contains "This program"`
- `Follow TCP stream`
- See the HTTP header data (`Host`)

### 5. What is the IP address and domain name of the compromised web site?
Answer: `http://www.kentroxarisma.com` (`173.254.80.53`)
Steps:
- See the referer url
- Search ip contains `http://www.kentroxarisma.com`

## LEVEL 2 QUESTIONS

### 1. What is the redirect URL that points to the exploit kit landing page?\
Answer: `genfaglobal.ga`
Steps:
- Search `ip contains "302 Found"`
- Pick the one with `seedradrivergy.co.vu` as Location
- Follow HTTP stream and see Host

2. What type of exploits (Java, Flash, PDF) were sent by the exploit kit?
### Answer: Flash, PDF
Steps:
- Search ip contains `seedradrivergy.co.vu`
- Follow HTTP Stream to see responses (Contains filetypes and names)

### 3. What is the MD5 hash of the malware payload?
Answer: `dd0a1445731256289b4e636ebbb4a00d`
Steps:
- Export Objects > HTTP
- Find the malware and save
- Check MD5 hash

### 4. What type of callback traffic is generated by the malware?
Answer: A lot of SMTP requests seen from infected machine (`Statistics > Protocol Hierarchy`)
